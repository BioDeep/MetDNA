1. What types of metabolomics data are supported by MetDNA?

    2. Ans. MetDNA only supports LC-MS based untargeted metabolomics data acquired from high resolution Time-of-Flight (TOF) and Orbitrap instruments. MS/MS data are required for the metabolite identification.MS/MS data from different data acquisition methods such as data dependent acquisition (DDA), data independent acquisition (DIA) or targeted MS2 acquisition are all supported by MetDNA. MetDNA do NOT support targeted metabolomics data, such as MRM and PRM. MetDNA do NOT support metabolomics data acquired from GC-MS and NMR.

  2. When I upload the data, why does “upload error” appear?

    3. Ans. Please make sure that first you choose and upload the positive mode data; after the uploading completes, then choose and upload negative mode data. You cannot simultaneously upload both positive and negative modes.

  3. I uploaded both positive and negative mode data, but I cannot choose “Both” in the setting of “Ionization Polarity” in Set Parameters page?

    4. Ans. Please make sure that two modes of your data are uploaded in the same project. The project name for both positive and negative modes must be the same.

  4. Can MetDNA support to process DIA based metabolomics data?

    5. Ans. Yes! You can use MS-DIAL to generate the MS1 peak table and MS2 data files in .msp format. MetDNA supports the import of MS2 data files in msp format.

  5. Does MetDNA support targeted metabolomics data?

    6. Ans. No!

  6. Does MetDNA support GC-MS data analysis?

    7. Ans. No!

  7. Does MetDNA support NMR data analysis?

    8. Ans. No!

  8. Which browser can I use?

    9. Ans. Currently, common web browsers such as Chrome, Edge and Firefox are supported. Internet Explorer (IE) didn't give sufficient test. We highly recommend Chrome.

  9. How to open the result file?

    10. Ans. The result of MetDNA analysis is a compressed archive format file, '.tar.gz', you can use 7-Zip to unzip this compressed file.

  10. When I upload the data, why does "File 'Sample Information' has error: The first column name is not 'sample.name'." appear? However, I am sure that the first column name of sample information is "sample.name".

    11. Ans. In some countries and regions (European and some French-speaking regions), the default separator is semicolon for .csv format files. So please check your sample information and MS1 peak table again to make sure they are separated by comma. You can open the sample information or MS1 peak table with notepad or other text editors to check whether they are separated by comma.
